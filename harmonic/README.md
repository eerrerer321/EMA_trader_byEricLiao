# 🔱 諧波型態交易策略（Harmonic Patterns）

> 一個**不重繪、誠實統計**的諧波型態策略，並與主專案的 EMA 趨勢策略**用同一筆資金混合**。

這個資料夾是獨立的研究模組。它的目標不是複製 TradingView 上那些「勝率 60-70%」的漂亮諧波指標，而是用**乾淨、可驗證**的方式回答一個問題：**諧波型態在去掉重繪與美化後，到底有沒有 edge？能不能跟趨勢策略疊加？**

---

## 一、為什麼這個版本不一樣

市面上多數諧波指標有兩大陷阱，這裡都刻意避開：

| 陷阱 | 一般指標 | 本策略 |
|---|---|---|
| **重繪 (repaint)** | 樞軸隨新價格變動，歷史圖上型態「事後」都完美命中 | 樞軸用 **N-bar 分形**，只在之後 N 根才確認；型態比例只用已確認的 X/A/B/C，D 點(PRZ)在 C 確認當下就固定 |
| **統計美化** | 「TP1 命中後不算輸」、9 組參數事後挑最佳 | 每個到達 PRZ 的型態**全部計入**（含失敗、含被掃停損），固定一組參數 |

**結論先講**：去掉水分後，無過濾的諧波真實實力只有 **PF 1.11、年化 1.3%**——遠不如那些宣稱的高勝率。但加上下面這個 edge 後，它變得真正有用。

---

## 二、核心 edge：順大勢（trend-aligned）

**諧波不該用來逆勢抄底摸頂，而該用來「在大趨勢方向上抓回調進場」。**

- 多方型態只在 `close > EMA200`（多頭環境）進場；空方型態只在 `close < EMA200`。
- 對照組「逆大勢抄底摸頂」PF 僅 **0.69（虧損）**，強力印證原理。

| 配置（2017-2026, ETH） | 進場數 | 勝率 | PF | 報酬 |
|---|---|---|---|---|
| 無過濾 | 146 | 30% | 1.18 | 32% |
| 逆大勢（抄底摸頂） | 67 | 19% | **0.69** ❌ | −21% |
| **順大勢（本策略）** | **61** | **38%** | **1.70** ✅ | **52%** |

### 樣本外驗證（非過擬合）
| 期間 | 進場 | 勝率 | PF |
|---|---|---|---|
| 樣本內 2017-2022 | 40 | 32% | 1.66 |
| **樣本外 2022-2026** | 21 | 48% | **1.83** |

樣本外 PF 不衰退反而更高 → 這是**真實的策略原理**，不是 curve-fitting。

---

## 三、最佳配置（皆經對照選定，避免過度優化）

| 項目 | 選擇 | 為什麼 |
|---|---|---|
| 樞軸半窗 | `PIVOT_N = 3` | 樣本/PF 最佳平衡（7 筆/年） |
| 型態 | Gartley / Bat / Butterfly / Crab（多空） | 經典五點型態 |
| 過濾 | 順大勢（close vs EMA200） | 核心 edge（見上） |
| 出場 | **單一 0.618 TP 全倉** | 實測分批止盈反而削弱高賠率（PF 1.70 vs 0.84） |
| sizing | 固定名目（20%×3，與趨勢一致） | 實測優於固定風險 |
| 同根 SL/TP | 虧損區停損先判、保本/獲利區止盈先判 | 公允、不自欺 |

> 教訓：分批止盈、固定風險、移動保本、額外濾網——這些「優化」**全都讓它變差**。最樸素的配置反而最好。

---

## 四、與趨勢策略混合：真正的 1+1>2

把 EMA 趨勢（85% 時間空倉）與順大勢諧波放在**同一筆資金**分時複用（趨勢優先、空倉時諧波補上、互斥持倉）：

| 同一筆資金 1000 | 報酬 | 最大回撤 | Sharpe | Calmar |
|---|---|---|---|---|
| 純趨勢 | 160% | −35.8% | 0.65 | 0.33 |
| 純諧波（順勢） | 52% | −12.1% | 0.48 | 0.41 |
| **混合** | **291%** | **−33.3%** | **0.83** | **0.51** |

**每個指標都改善**：報酬幾乎翻倍、回撤不增反降、Calmar +55%。關鍵是順大勢諧波本身就是**強策略 + 低相關（−0.10）**，所以疊加是真正的分散增益，而非「強策略 + 弱策略」的拖累。

## 六之二、多週期（小週期機會更多）

諧波在小週期出現更頻繁，但**雜訊也更多，樞軸大小要放大**才能守住 edge（2021-2026, ETH）：

| 週期 | 樞軸 N | 頻率 | PF | 樣本外 PF |
|---|---|---|---|---|
| 4h | 3 | 6/年 | 1.05* | — |
| **1h** | 3 | **23/年** | **1.26** | 1.09 |
| 30m | 8 | 9/年 | 1.53 | — |

\* 4h 全期(2017-2026) PF 1.70。**1h 是「機會 ×3.7 且 edge 守住」的甜蜜點**，故實盤預設 1h。

```bash
python fetch_data.py --timeframe 1h                       # 下載 1h
python harmonic_strategy.py --csv data/eth_1h.csv --start 2021-01-01 --pivot-n 3
```

## 六之三、實盤自動交易

### A. 諧波獨立實盤 — `harmonic_live_trading.py`
只跑諧波，限價掛在 PRZ + 交易所端 SL/TP。

```bash
python harmonic_live_trading.py                                  # DRY-RUN 觀察(預設 1h)
HARMONIC_TIMEFRAME=1h HARMONIC_PIVOT_N=3 HARMONIC_DRY_RUN=0 python harmonic_live_trading.py
```

### B. 混合實盤 — `run_mixed_live.py`（推薦）
**4h 趨勢 + 1h 諧波在同一個 Bybit 帳戶、同一筆資金**上跑，這才是回測 1+1>2 的實盤版：
- 互斥持倉（一次一倉）；趨勢訊號優先（諧波只是掛單就撤單讓趨勢進，已成交則趨勢讓步）；
- 趨勢空倉時諧波用閒置資金在 1h 找機會；
- 趨勢出場重用回測同款的固定+移動停損；諧波為 PRZ 限價 + 交易所 SL/TP；
- **趨勢腿倉位 = 20%×3 × 波動度目標係數**（與主程式同設定）。混合回測實證：報酬
  291%→530%（Binance 17-26）/ 99%→182%（Bybit 21-26）、Calmar 0.51→0.69 / 0.80→1.10，
  滾動視窗獲利率不變。諧波腿維持固定名目（加了反而讓滾動獲利率 78%→74%）。
- **趨勢腿進場帶「多頭擁擠」過濾**：BTC 幣本位 3 日平滑資金費率 > 0.01%/8h 時不開新
  多單（公開端點，讀取失敗自動旁路）。實證：該情境多單 PF 僅 0.81；過濾後 ETH Bybit
  21-26 PF 1.58→1.72、滾動獲利率 69→73%，BTC 18-26 報酬 330%→511%、MDD −32→−27%。
- **趨勢腿「深負費率加碼」**：BTC 費率 < −0.01%/8h（空方深度擁擠、軋空燃料）時新多單
  倉位 ×1.5，與波動度係數相乘後 cap 2.0（不放大曝險包絡）。實證：ETH Bybit 182.8%→188.7%、
  BTC 18-26 511%→637%，三資料集 MDD 完全不變；寬鬆版（<0 就加碼）實測有害已否決。

```bash
python run_mixed_live.py                       # DRY-RUN 觀察(預設)，先跑幾天看協調
HARMONIC_DRY_RUN=0 python run_mixed_live.py     # 實單
```
狀態存於 `mixed_state.json`，支援重啟恢復。

> ⚠️ 兩支實盤程式**皆未經實單測試**。首次實單請用最小金額，在 Bybit App 確認限價單、SL、TP 是否如預期。實盤與回測必有差異（成交價、滑價、樞軸確認延遲、4h/1h 邊界）。`run_mixed_live.py` 與主專案 `eth_strategy_4h_autotrading.py` **不要同時對同一帳戶運行**（會搶持倉）——混合實盤已自帶趨勢邏輯。

## 六、如何執行

```bash

python fetch_data.py --symbol ETH/USDT

# 2) 諧波策略單獨回測（含樣本內外、順大勢對照）
python harmonic_strategy.py --csv data/eth_4h.csv
python harmonic_strategy.py --csv data/eth_4h.csv --no-trend-filter   # 看關閉過濾的對照

# 3) 趨勢 + 諧波 同一筆資金混合
python run_mixed.py --csv data/eth_4h.csv
```

## 七、限制與提醒

- 這是**研究/回測模組**，不是實盤下單程式。實盤化需要自行接交易所 API（樞軸確認延遲、限價單掛 PRZ、停損停利掛單等）。
- 諧波頻率低（~7 筆/年），單獨用報酬有限；它的價值在**與趨勢混合降低回撤、提升 Calmar**。
- 所有數字基於 Binance ETH 現貨 2017-2026、手續費 0.06% + 滑價 0.02%、固定名目 20%×3。換標的/期間請重新驗證。
- 回測對「同一根 K 線同時觸及停損與止盈」採保守/公允假設，實盤滑價可能更差，請保留安全邊際。
